# Attachment 키 매핑 테이블

이 문서는 각 메시지 타입별 attachment 필드의 키 매핑을 정의합니다.

## 답장 메시지 (type 26)

| 키 | 설명 | 예시 |
|---|---|---|
| `src_message` | 원본 메시지 logId | `1234567890` |
| `logId` | 원본 메시지 logId (별칭) | `1234567890` |

## 반응 메시지 (type 70-79)

| 키 | 설명 | 예시 |
|---|---|---|
| `reaction` | 반응 타입 (숫자) | `0` (heart), `1` (thumbs_up) |
| `likeType` | 좋아요 타입 (별칭) | `0` |
| `emoType` | 이모지 타입 (별칭) | `0` |
| `message_id` | 대상 메시지 logId | `1234567890` |
| `target_id` | 대상 메시지 logId (별칭) | `1234567890` |
| `logId` | 대상 메시지 logId (별칭) | `1234567890` |
| `src_logId` | 대상 메시지 logId (별칭) | `1234567890` |

### 반응 타입 매핑

| 숫자 | 문자열 | 의미 |
|---|---|---|
| `0` | `heart` | ❤️ |
| `1` | `thumbs_up` | 👍 |
| `2` | `check` | ✅ |
| `3` | `surprised` | 😱 |
| `4` | `sad` | 😢 |

## 이미지 메시지 (type 2, 12, 27)

| 키 | 설명 | 예시 |
|---|---|---|
| `url` | 이미지 URL | `https://...` |
| `path` | 이미지 경로 | `/path/to/image.jpg` |
| `path_1` | 이미지 경로 (별칭) | `/path/to/image.jpg` |
| `thumbnailUrl` | 썸네일 URL | `https://...` |
| `xl`, `l`, `m`, `s` | 다양한 크기 URL | `https://...` |

## Feed 메시지 (type 12)

| 키 | 설명 | 예시 |
|---|---|---|
| `feedType` | Feed 타입 (숫자) | `6` (강퇴), `2` (퇴장) |
| `member` | 멤버 정보 | `{nickName: "...", userId: "..."}` |
| `kicker` | 강퇴한 사람 정보 | `{nickName: "...", userId: "..."}` |

### feedType 값

| 값 | 의미 |
|---|---|
| `1` | 초대 |
| `2` | 퇴장 |
| `4` | 오픈채팅 입장 |
| `6` | 강퇴 |
| `11` | 부방승급 |
| `12` | 부방강등 |
| `14` | 삭제 |
| `15` | 방장위임 |

## 참고

- 모든 attachment 필드는 복호화 후 JSON 형태로 파싱됩니다.
- 키는 대소문자를 구분합니다.
- 여러 별칭이 있는 경우, 우선순위는 위에서 아래 순서입니다.
